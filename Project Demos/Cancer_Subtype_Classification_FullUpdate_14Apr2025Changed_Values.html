--- 3. Data Partitioning ---¶

In [30]:
# Separate features (X) and target (y).
X = df.drop('Class', axis=1)
y = df['Class']

# Split the data into training (80%) and testing (20%) sets.
# Stratified splitting ensures that the class distribution is maintained.
X_temp, X_test, y_temp, y_test = train_test_split(
    X, y,
    test_size=0.2,
    stratify=y,
    random_state=42  # For reproducibility
)

--- 4. Feature Selection (Performed ONLY on Training Data) ---¶

In [31]:
# Step 1: Remove zero-variance features using VarianceThreshold.
vt = VarianceThreshold(threshold=0.2) # threshold=0.1,0.02,1...
X_train_vt = vt.fit_transform(X_temp)  # Fit ONLY on training data
X_test_vt = vt.transform(X_test)  # Transform test data

# Step 2: Select top features using SelectKBest and f_classif.
kbest = SelectKBest(f_classif, k=500) # k=300,500,600...
X_train_selected = kbest.fit_transform(X_train_vt, y_temp)  # Fit on training
X_test_selected = kbest.transform(X_test_vt)  # Transform test data

--- 5. Data Scaling ---¶

# Initialize Random Forest model with hyperparameters tuned by GridSearchCV
# Create StratifiedKFold object for cross-validation

# Define the parameter grid
param_grid = {
    'n_estimators': [100, 200, 300],
    'max_depth': [10, 15, 20],
    'min_samples_split': [2, 4],
    'min_samples_leaf': [1, 2]
}

# Initialize GridSearchCV
grid_search = GridSearchCV(
    estimator=RandomForestClassifier(random_state=42),
    param_grid=param_grid,
    scoring='balanced_accuracy',
    cv=cv,
    n_jobs=-1
)

# Fit the model
grid_search.fit(X_train_scaled, y_temp)

# Get the best parameters

The results and explanations are structured to be easily # understood and presented. # 1. Feature Selection (In $$78]) # VarianceThreshold(threshold=0): # threshold: This is the threshold for variance. Features with variance below this value are removed. # Safe Values: You can try different values, such as 0.01, 0.05, or 0.1. Higher values will remove more features. Setting it too high, though, might # remove useful information. # SelectKBest(f_classif, k=400): # k: This is the number of "best" features to select. # Safe Values: You can experiment with values like 200, 300, 500, or 600. The optimal value depends on your dataset. Too few features might underfit, # while too many might overfit. # f_classif: Is the scoring function, it would be dangerous to change this without understanding other types of metrics for your classifcations as that # may give you poor performing data. # 2. Model Training (Random Forest in $$80]) # Within param_grid: # n_estimators: The number of trees in the forest. # Safe Values: [100][200][300] are good starting points. More trees generally improve performance but increase training time. # max_depth: The maximum depth of the trees. # Safe Values: `` are reasonable ranges. Deeper trees can capture more complex relationships but are also more prone to overfitting. # min_samples_split: The minimum number of samples required to split an internal node. # Safe Values: `` are typical. Higher values can prevent overfitting. # min_samples_leaf: The minimum number of samples required to be at a leaf node. # Safe Values: [1] are common. Similar to min_samples_split, this helps prevent overfitting. # 3. Model Training (SVM in $$81]) # SVC(kernel='linear', class_weight='balanced', probability=True, random_state=42) # kernel: The kernel type to be used in the algorithm. It must be one of 'linear', 'poly', 'rbf', 'sigmoid', 'precomputed' or a callable. # Safe Values: linear, rbf # C: Regularization parameter. The strength of the regularization is inversely proportional to C. Must be strictly positive. # Safe Values: 1, 10, 100 # degree: Degree of the polynomial kernel function ('poly'). Must be non-negative. # Safe Values: 3, 4 # gamma: Kernel coefficient for 'rbf', 'poly' and 'sigmoid'. # 4. Model Training (Decision Tree in $$82]) # DecisionTreeClassifier(class_weight='balanced', random_state=42) # criterion: The function to measure the quality of a split. Supported strategies are “best” to choose the best split and “random” to choose the best # random split. # Safe Values: best, random # max_depth: The maximum depth of the tree.
